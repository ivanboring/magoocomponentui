# Authoring a Component

A component is a single source folder under `components/<category>/<name>/`. Run
`pnpm build` and the generator emits SDC, Drupal Code Component (Preact), React, Vue,
Storybook, and Drupal paragraph variants into `dist/<category>/<name>/`.

## Folder layout

```
components/<category>/<name>/
├── component.def.yml   # REQUIRED — prop/slot contract + variant class maps
├── template.html       # REQUIRED — tokenized-Tailwind markup + directives
├── metadata.yml        # REQUIRED — agent-facing catalog record (validated)
├── behavior.js         # optional — vanilla ES module: export default init(root, props)
├── examples/           # optional — <StoryName>.json arg payloads for Storybook
└── screenshots/        # generated by the screenshot pipeline (do not hand-write)
```

## 1. `component.def.yml` — the contract

```yaml
name: card-podcast            # machine name = folder name
props:
  title:   { type: string, required: true, title: Title, example: "Episode 42" }
  variant: { type: enum, values: [default, elevated], default: default }
  image:   { type: image, title: Cover }
  items:   { type: array, items: object, title: Chapters }
slots:
  footer:  { title: Footer, description: "Actions below the card." }
variants:                     # enum value → utility classes (see template {{ variant@class }})
  variant:
    default:  "bg-surface"
    elevated: "bg-surface shadow-raised"
```

Prop types: `string`, `html` (rich), `text` (multiline), `integer`, `boolean`, `enum`,
`link`, `image`, `array` (with `items`), `object`.

## 2. `template.html` — markup

Valid HTML5 using **tokenized Tailwind utilities** (never raw colors — see `theming.md`)
plus this directive vocabulary (full reference: `template-directives.md`):

```html
<article class="card-podcast rounded-card bg-surface p-(--space-card) {{ variant@class }}">
  <img data-if="image" class="rounded-card" src="{{ image }}" alt="{{ title }}">
  <h3 class="font-heading text-on-surface">{{ title }}</h3>
  <ol>
    <li data-for="item in items" class="text-on-surface-muted">{{ item.label }}</li>
  </ol>
  <slot name="footer"></slot>
</article>
```

- The root element carries `class="<name>"` as the **JS hook** (here `card-podcast`).
- Styling utilities and `__hook` classes are separate concerns — keep them so.

## 3. `behavior.js` — interactivity (optional, vanilla JS)

```js
export default function init(root, props) {
  const btn = root.querySelector(".card-podcast__play");
  const onClick = () => root.classList.toggle("is-playing");
  btn?.addEventListener("click", onClick);
  return () => btn?.removeEventListener("click", onClick); // cleanup
}
```

- Target component-scoped `__hook` classes (`.card-podcast__play`), not utility classes.
- **Read configuration from `root.dataset`, not `props`** — the portable SDC/preview/Storybook
  wrapper calls `init(root, {})`. Expose config as `data-*` attributes in the template.
- Implement real keyboard + ARIA here so the WCAG/keyboard metadata is truthful.

## 4. `metadata.yml` — catalog record

Validated against the schema (see `metadata-schema.md`). Required: `short_description`,
`lifecycle`, `categorization` (with `category`, `atomic_type`, `maturity`).

## Build & verify

```
pnpm build        # generate dist/ + catalog.json
pnpm test         # generator + schema unit tests
pnpm preview:dev  # browse the components across all 4 themes
pnpm storybook    # component workbench with theme toolbar
pnpm screenshots  # capture 4 themes × 4 breakpoints per component
```
